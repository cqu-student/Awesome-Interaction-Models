# 贡献指南

感谢你对 Awesome Interaction Models 的兴趣！我们很高兴能与你合作。

## 如何贡献

### 1. 提交新资源

我们欢迎您提交新的论文、文章和开源项目。请确保：

- **资源的质量**：提交的资源应该是该领域的重要贡献或广泛使用的项目。
- **正确的分类**：确保资源被放在正确的版区（Text/Image/Video）和正确的板块（论文/代码）。
- **清晰的描述**：为每个资源提供简洁的描述，说明其主要贡献或用途。
- **有效的链接**：确保所有链接都是有效的。

### 2. Pull Request 流程

1. Fork 该仓库
2. 创建一个新分支：`git checkout -b add-new-resources`
3. 编辑 `README.md` 文件
4. 提交更改：`git commit -m "Add [resource name]"`
5. 推送到你的 fork：`git push origin add-new-resources`
6. 创建 Pull Request，描述你添加的内容

### 4. 格式要求

请遵循以下格式：

**对于论文/文章：**
```
- [标题](链接) - 简洁的描述（1-2 句话）。
```

**对于开源项目：**
```
- [项目名称](GitHub 链接) - 简洁的描述和主要特性。
```

### 5. 排序规则

- 新资源应该按照相关性或热度排序
- 不同版区按照以下顺序：Text → Image → Video
- 每个版区内按照：论文/文章 → 开源代码

## 代码行为准则

我们承诺为所有参与者提供一个尊重和包容的环境。我们的社区成员应该：

- 使用尊重和包容的语言
- 尊重不同的观点和经验
- 接受建设性的批评
- 关注对社区最有利的事情
- 相互支持

## 许可证

通过贡献，你同意你的贡献将在相同的许可证下发布。
